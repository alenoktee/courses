# Руководство по запуску проекта

## Необходимые компоненты

### Для клиентской части (Angular)
- Node.js (версия 16.x или выше)
- npm (включен в установку Node.js)
- Angular CLI (версия 19.x)

### Для серверной части (ASP.NET Core)
- .NET SDK (версия 8.0)
- PostgreSQL (версия 14.x или выше)

## Настройка и запуск проекта

### Настройка базы данных
1. Установите PostgreSQL
2. Создайте базу данных PostgreSQL
3. Обновите строку подключения в `server/appsettings.json` при необходимости:
   ```json
   "ConnectionStrings": {
     "DefaultConnection": "Host=localhost;Port=5432;Database=postgres;Username=postgres;Password=postgres"
   }
   ```

### Настройка appsettings.json (/server)
1. Зайдите в файл appsettings.json
3. Обновите строку, добавив сюда свои данные из Google Cloud Console:
   ```json
   "Google": {
      "ClientId": "ЗАТЫЧКА",
      "ClientSecret": "ЗАТЫЧКА"
    }
   ```


### Запуск серверной части (ASP.NET Core)
1. Откройте терминал и перейдите в директорию `server`:
   ```
   cd server
   ```
2. Восстановите пакеты NuGet:
   ```
   dotnet restore
   ```
3. Примените миграции базы данных:
   ```
   dotnet ef database update
   ```
4. Запустите сервер:
   ```
   dotnet run
   ```
5. Сервер будет доступен по адресу: `https://localhost:7135` или `http://localhost:5231`

### Запуск клиентской части (Angular)
1. Откройте новый терминал и перейдите в директорию `client`:
   ```
   cd client
   ```
2. Установите зависимости:
   ```
   npm install
   ```
3. Запустите клиентское приложение:
   ```
   npm start
   ```
   или
   ```
   ng serve
   ```
4. Клиентское приложение будет доступно по адресу: `http://localhost:4200`

## Аутентификация Google
Для работы аутентификации через Google необходимо использовать существующие ключи в `appsettings.json` или создать свои собственные в [Google Cloud Console](https://console.cloud.google.com/).

## Проверка работоспособности
После запуска обеих частей проекта откройте браузер и перейдите по адресу: `http://localhost:4200`. Вы должны увидеть главную страницу приложения с возможностью входа через Google.

# Документация по разработке

## Структура проекта

Проект разделен на две основные части:
- `server/` - серверная часть на ASP.NET Core
- `client/` - клиентская часть на Angular

### Серверная часть (server/)

#### Основные директории и файлы:

1. **Controllers/**
   - `AuthController.cs` - контроллер для аутентификации
     - `GoogleLogin()` - обработка входа через Google
     - `Register()` - регистрация нового пользователя
     - `ExchangeCodeForTokenAsync()` - обмен кода авторизации на токен
     - `GetGoogleUserInfoAsync()` - получение информации о пользователе от Google
     - `GenerateJwtToken()` - генерация JWT токена
     - `HashPassword()` - хеширование пароля

2. **Models/**
   - Содержит модели данных для работы с базой данных
   - Определяет структуру таблиц и связи между ними

3. **Data/**
   - `AppDbContext.cs` - контекст базы данных
   - Содержит конфигурацию Entity Framework Core

4. **Migrations/**
   - Содержит миграции базы данных
   - Используется для управления схемой базы данных

5. **Program.cs**
   - Точка входа приложения
   - Настройка сервисов и middleware
   - Конфигурация CORS и аутентификации

6. **appsettings.json**
   - Конфигурация приложения
   - Строки подключения к базе данных
   - Настройки JWT и Google OAuth

### Клиентская часть (client/)

#### Основные директории и файлы:

1. **src/**
   - `app/` - основной код приложения
     - `components/` - переиспользуемые компоненты
     - `services/` - сервисы для работы с API
     - `models/` - модели данных
     - `guards/` - защитники маршрутов
     - `interceptors/` - HTTP-перехватчики
     - `app-routing.module.ts` - конфигурация маршрутизации
     - `app.module.ts` - корневой модуль приложения

2. **package.json**
   - Зависимости проекта
   - Скрипты для запуска и сборки

3. **angular.json**
   - Конфигурация Angular CLI
   - Настройки сборки и разработки

4. **proxy.conf.json**
   - Настройки прокси для разработки
   - Перенаправление запросов к API

## Архитектура приложения

### Серверная часть
- ASP.NET Core Web API
- Entity Framework Core для работы с базой данных
- JWT для аутентификации
- Google OAuth для социальной аутентификации
- PostgreSQL как основная база данных

### Клиентская часть
- Angular 19
- TypeScript
- RxJS для работы с асинхронными операциями
- Angular Material для UI компонентов
- JWT для хранения токена аутентификации

## Процесс разработки

1. **Настройка окружения**
   - Установка .NET SDK 8.0
   - Установка Node.js 16.x
   - Установка PostgreSQL 14.x

2. **Разработка серверной части**

3. **Разработка клиентской части**
